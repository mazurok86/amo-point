# Этап 2 — JS: динамическая видимость полей по типу

Решение НЕ интегрируется в Laravel-приложение — это standalone-файл/сниппет под чужую страницу `http://test.amopoint-dev.ru/testzz/testlist.html`.

## Поставка

Два варианта в одном файле:
- `public/testzz/testlist.js` — подключается тегом `<script src="...">` (для демонстрации можно положить рядом с проектом).
- Минифицированный one-liner для копипаста в DevTools Console.

В чистовике для сдачи: одиночный файл `testlist.js` + копия как сниппет в README.

## Алгоритм (выбранный)

1. На `DOMContentLoaded` находим селект `select[name="type"]`.
2. Слушаем `change` через делегирование на `document` (надёжно при пересборке DOM).
3. На каждое срабатывание:
   - `value = select.value.trim().toLowerCase()`;
   - перебираем `document.querySelectorAll('input[name], textarea[name], select[name]:not([name="type"])')`;
   - для каждого поля: `match = field.name.toLowerCase().includes(value)`;
   - **скрываем/показываем не сам input, а его обёртку** — `field.closest('.form-row, label, .field, p, div')` (иначе остаются висящие label).
4. Запускаем один раз сразу после загрузки — чтобы начальное состояние было консистентно.
5. Если `value === ''` — показываем все поля (sane default).

## Что считать «содержит»

Приоритетный вариант — `String.prototype.includes` (substring). Почему:
- ТЗ дословно говорит «есть значение элемента списка» → подстрока, не точное совпадение.
- Покрывает кейсы вида `phone_mobile`, `phone_home`, где `value="phone"`.

Альтернатива — `name.split(/[_\-.]/).includes(value)` (по разделителям). Точнее, но рискует не совпасть при склейках без разделителей. Указать в README.

## Альтернативы (с обоснованием отказа)

| Подход | Почему не выбран |
|---|---|
| **CSS attribute selector** + динамический `<style>`: `input[name*="x"] { display: revert } ; input[name] { display: none }` | Лаконично, но прячет только сам input, не родительский label/контейнер. Селектор `:has()` решает, но поддержка хуже. |
| **MutationObserver** на форму | Нужен только при динамическом добавлении полей. На статической форме оверкилл. |
| **jQuery** `$('input[name*=...]').show()` | Лишние ~30 КБ ради десятка строк нативного JS. |
| **Alpine / Vue / React** | Требует переписывать разметку формы. ТЗ просит подключаемый сниппет к существующей странице. |
| **Жёсткая мапа `{type: [имена полей]}`** | Не масштабируется, противоречит ТЗ («поля, в name которых есть значение»). |
| **Скрытие через `hidden` атрибут vs `style.display='none'`** | Использую `hidden` — семантично, не конфликтует с inline-стилями страницы. |
